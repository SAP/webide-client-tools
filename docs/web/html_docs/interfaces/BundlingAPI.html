internal

internal: any
  • Defined in api.d.ts:184

Internal utilities to construct the "bundleFeature" flow. These are exposed as unofficial APIs to enable the creation of advanced flows by end users. However, they receive the same level of support as the official APIs.

bundleFeature

  • bundleFeature(target: string, options?: { bundler?: "webpack" | "requirejs"; cleanOutDir?: boolean; enableCaching?: boolean; javaScriptOpts?: { ignore?: string | string[]; ignoreValidations?: boolean; optimizeOptions?: Object }; metadataOpts?: { pluginsPrefix?: string }; outDir?: string; webpackConfig?: Configuration }): Promise<{ outDir: string }>
    • Defined in api.d.ts:89

    Bundles an SAP Web IDE feature, including JS sources, “i18n” files, and JSON metadata. It does not include the bundling of any SAPUI5 resources. Use tools provided by SAPUI5 to bundle those.

    Returns a promise with an object. The object properties: outDir: output directory containing the bundled files. The output directory's name may not be known in advance as it may include a timestamp when the caching option is enabled. By exposing the exact name, custom post-processing steps may be more easily implemented by the user.

    Parameters

    • target: string
    • Optional options: { bundler?: "webpack" | "requirejs"; cleanOutDir?: boolean; enableCaching?: boolean; javaScriptOpts?: { ignore?: string | string[]; ignoreValidations?: boolean; optimizeOptions?: Object }; metadataOpts?: { pluginsPrefix?: string }; outDir?: string; webpackConfig?: Configuration }
      • Optional bundler?: "webpack" | "requirejs"

        Which JavaScript bundler to use. Note that each of the bundlers have their configuration key.

        • "javascriptOpts" for requirejs.
        • "webpackConfiguration" for webpack.
      • Optional cleanOutDir?: boolean

        Should the output directory be cleaned prior to the bundling? This is enabled by default. If you have custom code that creates additional packaged artifacts in this directory, disabling this option should be considered. Default value is "true".

      • Optional enableCaching?: boolean

        Bundle into "outDir/[VERSION]_[TIMESTAMP]/" instead of "outDir/" and create a wrapper "package.json" file in the "outDir/package.json" file to enable caching support. Default value is "true".

      • Optional javaScriptOpts?: { ignore?: string | string[]; ignoreValidations?: boolean; optimizeOptions?: Object }

        SubOptions for the javaScript bundling phase. The are only relevant when using the require.js optimizer for bundling.

        • Optional ignore?: string | string[]

          Ignored glob patterns, by default all JS resources (recursively) in the directory of the "plugin.json" file are bundled. However, some special edge cases may require exclusion. For details about valid pattern syntax, @link https://github.com/isaacs/node-glob.

        • Optional ignoreValidations?: boolean

          Will only log validation errors on the bundled artifacts instead of throwing an error that rejects the promise. Enable it only if you are an advanced user and understand the ramifications.

          link

          {https://github.com/SAP/webide-client-tools/blob/master/FAQ.md#VALIDATE

        • Optional optimizeOptions?: Object

          Custom configurations for the "require.js" optimizer. For details, @link http://requirejs.org/docs/optimization.html#options.

      • Optional metadataOpts?: { pluginsPrefix?: string }

        SubOptions for the JSON file metadata bundling phase.

        • Optional pluginsPrefix?: string

          Prefix to apply to all plugin paths, such as "quickstart" --> "prefix/quickstart".

      • Optional outDir?: string

        Output directory for the bundle. Default value is "dist".

      • Optional webpackConfig?: Configuration

        The webpack configuration
